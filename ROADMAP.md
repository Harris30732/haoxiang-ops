## Phase 2：狀態管理 — 人員 × 店鋪

### 目標：精確追蹤每個人、每家店的狀態

#### 人員狀態
- [ ] 上班中（哪間店）
- [ ] 下班
- [ ] 接班中

#### 店鋪狀態
- [ ] 未開攤
- [ ] 開攤中（記錄開攤時間 + 開攤人員）
- [ ] 營業中
- [ ] 已關攤（記錄關攤時間 + 關攤人員）

#### 營業流程（不能反人類行為）
```
開攤（1人）→ 營業中
開攤（2人同時）→ 營業中（雙人開攤）
上班中 + 接班 → 換班完成（前一人退勤）
關攤 → 結算 → 店鋪關閉
```

#### 防呆規則
- 沒開攤不能打卡上班
- 沒人在店裡不能關攤（至少要有關攤人）
- 已關攤不能再打卡
- 同一人不能在兩間店同時上班

## Phase 4：AI 零錢結算（未來）

### 目標：關攤時用 AI 判斷零錢數量

#### 流程
```
關攤人員 → 輸入鈔票金額
         → 拍零錢盒照片（格子數的零錢盒）
         → AI 辨識零錢總額
         → 系統結算：鈔票 + 零錢 = 當日業績
```

#### 零錢基礎
| 店鋪 | 標準零錢數 | 說明 |
|------|-----------|------|
| 蘆洲店 | $2,150 | 固定放置 |
| 新莊店 | $2,350 | 固定放置 |

#### 零錢結轉規則
- 如果當天沒有回補到標準數，以實際結算數字當作隔天的零錢基礎
- 每日業績 = 鈔票 + 零錢實際數 - 前一天零錢基礎
