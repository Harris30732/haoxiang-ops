## 設計原則

### 紀錄模式
- **每店每天 1 筆**：開攤時間、關攤時間、營業時數、所有人員工時明細、業績
- **每人每天 1 筆**：上班時間、下班時間、工時、所屬店鋪

### 角色分離
- **員工**：只需要知道「打卡」，系統自動判斷開攤/上班/下班/關攤
- **管理員**：完整後台 — 報表、排班、修正紀錄、設定店鋪

### 核心狀態邏輯
- 接班 ≠ 換班：新人來了舊人不一定走，可同時 2-3 人在店
- 人員陸續下班，最後 1 人走時才關攤結算
- 「打卡」一個指令走天下，系統智慧判斷意圖

## 雙狀態機設計

### 店鋪狀態機
```
CLOSED → [開攤] → OPEN → [關攤] → SETTLING → [結算完成] → CLOSED
                    ↑↓
              人進人出都維持 OPEN
```

### 人員狀態機（兩個維度獨立）
- work_status（勤務）：OFF_DUTY / ON_DUTY
- current_step（對話）：IDLE / WAIT_STORE / WAIT_PHOTO / WAIT_REVENUE / WAIT_CLOSE_REVENUE

### 「打卡」智慧判斷
| 勤務狀態 | 店鋪狀態 | 系統行為 |
|---------|---------|---------|
| OFF_DUTY + 沒店開 | 全 CLOSED | → 開攤流程（選店→拍照→開攤）|
| OFF_DUTY + 有店開 | 有 OPEN | → 上班流程（選已開的店→拍照）|
| ON_DUTY + 店裡 >1 人 | OPEN | → 下班流程（拍照→記錄工時）|
| ON_DUTY + 只剩自己 | OPEN | → 問「下班」還是「關攤結算」|

### 防呆規則
- 沒開攤不能打卡上班
- 已關攤不能再打卡（會進入重新開攤流程）
- 同一人不能在兩間店同時上班
- 關攤時店裡必須只剩 1 人
- 開攤必須先選店 + 拍照

## 技術決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| 狀態儲存 | Google Sheets | 已有基礎，管理員可直接看 |
| Workflow 版本 | build_v18.js（不動 v17）| 安全回退 |
| n8n 並發 | concurrency=1 | 避免同時打卡覆蓋 |
| AI 零錢辨識 | Gemini Vision | 免費額度 |
| 外送平台 | n8n HTTP Request | 各平台 API |
| 角色判斷 | 員工管理表「角色」欄位 | 簡單有效 |
